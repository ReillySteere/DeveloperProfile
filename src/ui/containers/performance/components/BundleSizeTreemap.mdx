# BundleSizeTreemap

A treemap visualization for JavaScript bundle analysis, showing relative module sizes with interactive tooltips.

## Overview

`BundleSizeTreemap` displays bundle composition as a proportional treemap, helping developers identify large dependencies and optimization opportunities. Each rectangle represents a module, sized by its contribution to the total bundle.

## Usage

```tsx
import { BundleSizeTreemap } from 'ui/containers/performance/components/BundleSizeTreemap';

<BundleSizeTreemap
  modules={[
    {
      name: 'react-dom',
      path: 'node_modules/react-dom',
      size: 120000,
      gzippedSize: 38000,
      isInitial: true,
    },
    {
      name: 'lodash',
      path: 'node_modules/lodash',
      size: 72000,
      gzippedSize: 25000,
      isInitial: true,
    },
    {
      name: 'app',
      path: 'src/index.tsx',
      size: 45000,
      gzippedSize: 12000,
      isInitial: true,
    },
  ]}
  totalSize={500000}
  gzippedSize={150000}
/>;
```

## Props

| Prop          | Type             | Required | Description                            |
| ------------- | ---------------- | -------- | -------------------------------------- |
| `modules`     | `BundleModule[]` | Yes      | Array of modules with size data        |
| `totalSize`   | `number`         | Yes      | Total uncompressed bundle size (bytes) |
| `gzippedSize` | `number`         | Yes      | Total gzipped bundle size (bytes)      |

### BundleModule Interface

```tsx
interface BundleModule {
  name: string; // Display name (e.g., "react-dom")
  path: string; // Full path to module
  size: number; // Uncompressed size in bytes
  gzippedSize: number; // Gzipped size in bytes
  isInitial: boolean; // Part of initial bundle?
}
```

## Features

### Size Summary Header

Displays total bundle statistics:

- Total size (formatted as KB/MB)
- Gzipped size with compression ratio
- Number of modules analyzed

### Interactive Treemap

- **Proportional sizing**: Rectangle area corresponds to module size
- **Hover tooltips**: Shows module name, path, and exact sizes
- **Color coding**: Distinct colors for easy differentiation

### Responsive Container

Uses Recharts' `ResponsiveContainer` to fill available width while maintaining aspect ratio.

## Accessibility

- **Card structure**: Proper semantic container with title
- **Tooltip content**: Size information accessible on hover/focus
- **Color palette**: Colors are supplemented with labels in tooltips

## Design Rationale

### Treemap Visualization

Treemaps excel at showing hierarchical data proportionally. For bundle analysis:

- Large modules are immediately visible (big rectangles)
- Small modules naturally fade into the background
- Relative comparisons are intuitive

### Custom Content Renderer

The `CustomContent` component handles edge cases:

- Hides labels when rectangles are too small (< 40px width or < 20px height)
- Applies rounded corners for visual polish
- Uses CSS variables for theme-aware stroke colors

### Color Palette

## Size Formatting

The `formatSize` helper provides human-readable sizes:

```tsx
function formatSize(bytes: number): string {
  if (bytes < 1024) return `${bytes} B`;
  if (bytes < 1024 * 1024) return `${(bytes / 1024).toFixed(1)} KB`;
  return `${(bytes / (1024 * 1024)).toFixed(2)} MB`;
}
```

Bundle data is generated by the build-time analysis script:

- `scripts/analyze-bundle.js` runs webpack-bundle-analyzer
- Results are saved to `public/data/bundle-analysis.json`
- The performance container fetches this at runtime
